# Clinical Governance Framework

## 1. Decision Support Posture

REGENOVA-Intel operates exclusively as a **Clinical Decision Support (CDS) tool**. The system:

- **Does not** diagnose, prescribe, or recommend treatments autonomously
- **Does not** create a clinician-patient relationship
- **Does** synthesise evidence to support informed clinical decision-making
- **Does** flag safety concerns for clinician review
- **Does** attribute all statements to source evidence with tier ratings

All API responses carry the header `X-Decision-Support-Only: true`.

---

## 2. Evidence Hierarchy Policy

Evidence is classified into five tiers. The system weights retrieved evidence accordingly:

| Tier | Description | Weight | Examples |
|------|-------------|--------|---------|
| 1 | Peer-reviewed published research | 1.00 | PubMed, RCTs, systematic reviews |
| 2 | Clinical protocol documents | 0.85 | Protocol PDFs, clinical guidelines |
| 3 | Educational / practitioner content | 0.65 | Courses, reputable web sources |
| 4 | Community practitioner discussion | 0.40 | Forums, Skool community |
| 5 | Anecdotal / unverified | 0.15 | User testimonials |

**Policy:** No Tier 4 or 5 evidence alone supports a clinical recommendation. Tier 4/5 sources are used only to supplement Tier 1-3 evidence.

## 3. Safety Rule Catalogue

### Rule SR-001: Pregnancy / Breastfeeding Caution
- **Trigger:** Patient case indicates pregnancy or breastfeeding, AND query involves any peptide
- **Severity:** `critical`
- **Action:** Block reconstitution guidance; add mandatory specialist referral language
- **Rationale:** Safety profiles of most research peptides in pregnancy are unknown

### Rule SR-002: Oncology History Caution
- **Trigger:** Patient case indicates current or past malignancy, AND query involves growth-signaling peptides (IGF-1, CJC-1295, ipamorelin, GHK-Cu, TB-500, BPC-157)
- **Severity:** `warning`
- **Action:** Highlight theoretical growth-promotion risk; require oncologist consultation
- **Rationale:** IGF-1 pathway stimulation carries theoretical cancer risk

### Rule SR-003: Missing Baseline Labs Warning
- **Trigger:** Patient case has empty `baseline_labs` field, AND query is clinical
- **Severity:** `info`
- **Action:** Recommend baseline labs before initiating peptide protocol
- **Rationale:** Labs (IGF-1 levels, LFTs, HbA1c) needed for safe prescribing

### Rule SR-004: Polypharmacy Interaction Caution
- **Trigger:** Patient case lists ≥3 medications
- **Severity:** `info`
- **Action:** Flag for pharmacist review; note limited peptide-drug interaction data
- **Rationale:** Limited published interaction data for most peptides

### Rule SR-005: Reconstitution Guidance Gate (planned)
- **Trigger:** `include_reconstitution=true` in request AND role ≠ `clinician`
- **Severity:** `critical` (access block)
- **Action:** Return 403; log access attempt
- **Rationale:** Reconstitution guidance appropriate only for licensed prescribers

## 4. Contraindication Logic

Growth-signaling peptides (BPC-157, TB-500, GHK-Cu, IGF-1 variants, CJC-1295, ipamorelin, GHRP-6, GHRP-2, Hexarelin, MK-677) carry the following general contraindication flags:

| Condition | Severity | Notes |
|-----------|----------|-------|
| Active malignancy | `critical` | All growth-signaling peptides |
| Pregnancy | `critical` | All peptides (insufficient safety data) |
| Breastfeeding | `critical` | All peptides (insufficient safety data) |
| Proliferative diabetic retinopathy | `warning` | IGF-1 pathway peptides |
| History of malignancy < 5yr | `warning` | Growth-signaling peptides |
| Pre-diabetes / T2DM uncontrolled | `info` | Monitor insulin sensitivity |

## 5. Role-Based Access

| Role | Chat | Reconstitution Guidance | Ingest Trigger | Source Browser |
|------|------|------------------------|----------------|----------------|
| `anonymous` | ✗ | ✗ | ✗ | ✗ |
| `researcher` | ✓ (no clinical context) | ✗ | ✗ | ✗ |
| `clinician` | ✓ | ✓ (feature flag) | ✗ | ✗ |
| `admin` | ✓ | ✓ | ✓ | ✓ |

Roles are encoded in the JWT `role` claim. API key auth grants `admin` role.

## 6. Audit Logging Requirements

Every `/chat` request must produce an audit log entry containing:

```json
{
  "event": "chat_query",
  "request_id": "uuid",
  "timestamp": "ISO8601",
  "role": "clinician",
  "query_hash": "sha256(query)",
  "safety_flags_triggered": ["SR-001"],
  "confidence": 0.72,
  "source_tiers_used": [1, 2, 3],
  "latency_ms": 1250,
  "session_id": "optional"
}
```

**Note:** Query text is hashed before logging to avoid PII storage. Raw query is never persisted in logs.

## 7. Disclaimer Language

**Standard disclaimer** (appended to all responses):

> *This response is generated by REGENOVA-Intel, a clinical decision support tool. It does not constitute medical advice, diagnosis, or treatment. All information should be reviewed and validated by a qualified healthcare professional before clinical application. Evidence ratings reflect the quality of retrieved sources, not a guarantee of clinical efficacy or safety.*

**Escalated disclaimer** (when critical safety flags present):

> *⚠️ IMPORTANT SAFETY NOTICE: This query has triggered one or more critical safety flags. The information below must not be used to guide clinical decisions without specialist consultation. If a patient is pregnant, breastfeeding, or has a history of malignancy, consult with the appropriate specialist before considering any peptide intervention.*

## 8. Regulatory Considerations

### CE/FDA Note

REGENOVA-Intel is **not a regulated medical device** under:
- EU MDR 2017/745 (Medical Device Regulation)
- US FDA 21 CFR Part 820 (Quality System Regulation)

It is classified as a **clinical decision support software (CDSS)** tool that:
- Does not acquire, process, or analyse physiological data
- Does not drive or influence decisions through automated means without clinician review
- Falls within the "low-risk CDS" category under FDA guidance (Dec 2017)

**If this system is deployed in a regulated clinical setting, a full software risk assessment under IEC 62304 and IEC 62366 must be conducted.**
